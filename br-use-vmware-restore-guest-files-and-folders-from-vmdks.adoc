= Ripristinare file e cartelle da dischi virtuali
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Ripristina file o cartelle da un disco virtuale alla macchina virtuale originale o a un'altra macchina virtuale. Se preferisci non collegare il disco virtuale alla macchina virtuale originale, puoi collegarlo invece a una macchina virtuale proxy.

.Prima di iniziare
* Esamina i prerequisiti e le limitazioni in link:br-use-vmware-restore-guest-files-and-folders-overview.html["Requisiti e limitazioni durante il ripristino di file e cartelle guest"].
* Per ripristinare file e cartelle utilizzando una VM proxy, assicurarsi che la VM proxy sia già configurata prima di avviare il processo di ripristino di file e cartelle.
* È necessario creare le credenziali per il disco virtuale di origine e la macchina virtuale di destinazione in NetApp Backup and Recovery prima di poter ripristinare file e cartelle. NetApp Backup and Recovery utilizza queste credenziali per autenticarsi con il disco virtuale e la macchina virtuale di destinazione durante il ripristino di file e cartelle.


.Informazioni su questo compito
Le prestazioni del ripristino di file o cartelle dipendono da due fattori: le dimensioni dei file o delle cartelle da ripristinare e il numero di file o cartelle da ripristinare. Il ripristino di un gran numero di file di piccole dimensioni potrebbe richiedere più tempo del previsto rispetto al ripristino di un numero ridotto di file di grandi dimensioni, a parità di dimensioni complessive del set di dati da ripristinare.

È possibile eseguire un ripristino su una macchina virtuale remota (noto come ripristino tra macchine virtuali), ma i sistemi operativi di origine e di destinazione devono essere identici.


IMPORTANT: Su una VM è possibile eseguire solo un'operazione di collegamento o ripristino alla volta. Non è possibile eseguire operazioni di collegamento o ripristino parallele sulla stessa VM.


IMPORTANT: Con la funzione di ripristino di file e cartelle, è possibile visualizzare e ripristinare file di sistema e nascosti e visualizzare file crittografati. Non sovrascrivere un file di sistema esistente né ripristinare file crittografati in una cartella crittografata. Durante l'operazione di ripristino, gli attributi nascosto, di sistema e crittografato dei file guest non vengono mantenuti nel file ripristinato. La visualizzazione o l'esplorazione di partizioni riservate potrebbe causare un errore.

[role="tabbed-block"]
====
.Collega il file VMDK alla macchina virtuale originale per ripristinare file e cartelle
--
Ripristina i file e le cartelle guest da un disco virtuale alla macchina virtuale originale (source).

.Passi
. Dal menu NetApp Backup and Recovery , selezionare *Ripristina*.
. Seleziona *VMware* dall'elenco dei carichi di lavoro in alto a destra della pagina.
. Nell'elenco delle macchine virtuali, seleziona l'icona Azioni image:../media/icon-action.png["Opzione Azioni"] per una VM che contiene i file che desideri ripristinare.
. Seleziona *Ripristina file e cartelle*.
. Selezionare uno snapshot da cui effettuare il ripristino, quindi selezionare *Avanti*.
. Selezionare la posizione dello snapshot da cui effettuare il ripristino. Se si sceglie una posizione secondaria, selezionare lo snapshot secondario dall'elenco.
. Selezionare *Avanti*.
. Dall'elenco, scegli un disco virtuale che contenga i file e le cartelle da ripristinare e poi seleziona *Avanti*.
. Nella pagina _Dettagli della macchina virtuale guest_, procedi come segue:
+
.. Nella sezione *Dettagli della macchina virtuale guest*, collega il disco virtuale alla VM originale selezionando *Original virtual machine*.
.. Nella sezione *Credenziali della macchina virtuale guest*, se non hai ancora memorizzato una credenziale per il disco virtuale di origine e la macchina virtuale di destinazione, seleziona *Aggiungi credenziali*, inserisci le informazioni sulle credenziali di Windows o Linux e seleziona *Aggiungi*.
+

NOTE: Le macchine virtuali di origine e di destinazione devono utilizzare la stessa famiglia di sistemi operativi; le versioni del sistema operativo possono essere diverse.

.. Dall'elenco, scegli una credenziale di macchina virtuale da utilizzare.
.. Selezionare *Avanti*.
+
NetApp Backup and Recovery collega il disco virtuale alla macchina virtuale originale e visualizza tutti i file e le cartelle, inclusi quelli nascosti. Per i sistemi guest Windows, assegna una lettera di unità a ogni partizione, comprese le partizioni riservate al sistema.

+
È possibile utilizzare l'icona della lente d'ingrandimento (ricerca) vicino al riquadro di esplorazione file per cercare file e cartelle. La corrispondenza di pattern non è supportata, ma è possibile cercare file o cartelle in base a un sottoinsieme del nome o dell'estensione.



. Seleziona i file o le cartelle da ripristinare.
+
I file e le cartelle selezionati per il ripristino sono elencati nel riquadro destro dello schermo.

. Selezionare *Avanti*.
. Nella sezione "Ripristina nel percorso", immettere il percorso della destinazione VM e la posizione del file system in cui verranno ripristinati i file selezionati:
+
** Per i guest Windows, immettere il percorso di condivisione UNC:
+
*** Esempio di percorso IPv4: `\\10.60.136.65\c$`
*** Esempio di percorso IPv6: `\\fd20-8b1e-b255-832e--61.ipv6-literal.net\C\restore`


** Per i guest Linux, immettere il percorso del filesystem locale o l'indirizzo e il percorso del guest remoto:
+
*** Esempio di percorso locale: `/home/user/documents/file.txt`
*** Esempio di percorso IPv4: `10.60.136.65:/home/user/restore/`
*** Esempio di percorso IPv6: `fd20-8b1e-b255-832e--61.ipv6-literal.net:/home/user/restore/`




+
Se sono già presenti file con lo stesso nome, puoi scegliere di sovrascriverli o ignorarli.

. Nella sezione "Opzioni post-ripristino", è possibile disconnettere la sessione guest al termine del ripristino abilitando l'impostazione *Disconnetti sessione guest al termine del ripristino*. Questa operazione scollega il disco virtuale e smonta il datastore. Ciò significa che sarà necessario riconnettere la sessione guest prima di eseguire ulteriori operazioni di ripristino di file e cartelle.
. Selezionare *Ripristina*.
+
È possibile visualizzare l'avanzamento del ripristino nella pagina Monitoraggio processi.

.Collega il VMDK a una VM proxy per ripristinare file e cartelle
--
Utilizza una VM proxy (una VM diversa sullo stesso vCenter dell'originale) per ripristinare file e cartelle guest quando non desideri collegare il disco virtuale alla VM originale.

.Passi
. Dal menu NetApp Backup and Recovery , selezionare *Ripristina*.
. Seleziona *VMware* dall'elenco dei carichi di lavoro in alto a destra della pagina.
. Nell'elenco delle macchine virtuali, seleziona l'icona Azioni image:../media/icon-action.png["Opzione Azioni"] per una VM che contiene i file che desideri ripristinare.
. Seleziona *Ripristina file e cartelle*.
. Selezionare uno snapshot da cui effettuare il ripristino, quindi selezionare *Avanti*.
. Selezionare la posizione dello snapshot da cui effettuare il ripristino. Se si sceglie una posizione secondaria, selezionare lo snapshot secondario dall'elenco.
. Selezionare *Avanti*.
. Dall'elenco, scegli un disco virtuale che contenga i file e le cartelle da ripristinare e poi seleziona *Avanti*.
. Nella pagina _Dettagli della macchina virtuale guest_, procedi come segue:
+
.. Nella sezione *Dettagli della macchina virtuale guest*:
+
... Collega il disco virtuale a una VM proxy selezionando *Macchina virtuale proxy*.
... Scegliere un vCenter, data center e datastore in cui si trova la VM proxy dagli elenchi.
... Seleziona una VM dall'elenco da utilizzare come VM proxy. Il disco virtuale verrà collegato a questa VM. La VM proxy selezionata viene visualizzata nel riquadro di destra.


.. Nella sezione *Credenziali della macchina virtuale guest*, se non hai ancora memorizzato una credenziale per il disco virtuale di origine e la macchina virtuale di destinazione, seleziona *Aggiungi credenziali*, inserisci le informazioni sulle credenziali di Windows o Linux e seleziona *Aggiungi*.
+

NOTE: Le macchine virtuali di origine e di destinazione devono utilizzare la stessa famiglia di sistemi operativi; le versioni del sistema operativo possono essere diverse.

.. Dall'elenco, scegli una credenziale di macchina virtuale da utilizzare.
.. Selezionare *Avanti*.
+
NetApp Backup and Recovery collega il disco virtuale alla VM proxy e visualizza tutti i file e le cartelle, inclusi quelli nascosti. Per i guest Windows, assegna una lettera di unità a ogni partizione, comprese le partizioni riservate al sistema.

. Selezionare *Avanti*.

** Per i guest Windows, immettere il percorso di condivisione UNC:

*** Esempio di percorso IPv6: `\\fd20-8b1e-b255-832e--61.ipv6-literal.net\C\restore`
